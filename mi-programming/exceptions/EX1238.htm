Derived Field Operation Error

During derived field operations, one of a variety of derived field operational errors occurred. The field number designates the relative location of the field as specified when creating the cursor or data space index. Field number equal 1 is the first field in the field location. The field location is described by the field location exception data.

The error type is identical with those for the conversion mapping error  (hex 1201) exception with the addition of:
Hex 000B = An attempt has been made to divide by zero on a fixed-point divide operation.
Hex 000E = An attempt has been made to divide by zero on a floating-point divide operation.

The join cursor ordinal positions area is meaningful only on operations with a join cursor and is zero otherwise. It will be zero for group-by selection. Each ordinal number occupies 4 bytes. All current ordinal numbers associated with a join cursor are returned.

The ordinal entry number will contain a binary 0 value if the exception occurs during inserts of new entries into a data space or the error occurs while processing default values during a join operation.

The ordinal entry number and data space number will contain a value of binary 0 when the error occurs during group-by derived field operations.

Constraint identifier length contains the number of bytes in the constraint identifier. This field will contain a value of binary 0 unless the exception occured during the enforcement of a check constraint.

Constraint identifier contains a character scalar value identifying the check constraint that is responsible for the exception being signalled.
