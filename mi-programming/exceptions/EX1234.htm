The join cursor ordinal positions area is meaningful only on operations with a join cursor and is zero otherwise. It will be zero for group-by selection. Each ordinal number occupies 4 bytes. All current ordinal numbers associated with a join cursor are returned.

The ordinal entry number will contain a binary 0 value if the exception occurs while processing default values during a join operation.

The underlying mapping error field gives the error type encountered when the intermediate buffer was mapped. The values are given in the documentation for the derived field operation error  (hex 1238) exception, which also refers to the conversion mapping error  (hex 1201) exception. If the value is zero, the intermediate buffer mapping encountered no error.

The error type values are identical with those received for the data space index with non-user exit selection routine build termination  (hex 1233) exception.

The field number designates the relative location of the field as specified when creating the cursor, index, or data space. Field number equal 1 is the first field in the field location. A field number equal 0 designates there is no exception data for this operand.
