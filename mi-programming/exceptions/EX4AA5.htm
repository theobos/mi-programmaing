Java Virtual Machine Terminated

The Java Virtual Machine (JVM) has been terminated because a Java program called the java.lang.System.exit method, because a thread required for operation of the JVM was terminated or because the JVM detected an unrecoverable error.

All secondary threads attached to the JVM have been sent terminate thread interrupts. The threads will be detached from the JVM and then terminated as soon as the interrupt can be processed.

This exception is signalled only to the initial thread of the process and only if it is attached to the terminated JVM.

JVM resources are deallocated.

Reason code hex values and their meanings are as follows:

Code

Meaning

04

A thread has terminated which is required for operation of the JVM.
