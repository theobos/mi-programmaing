The class name identifies the name of the class, if any, being processed when the error being reported occurred. If the full class name exceeds 256 characters, the rightmost 256 characters are provided. This field is right-padded with blanks as necessary.

The method name identifies the name and descriptor of the method, if any, being processed when the error being reported occurred. If the full method name and descriptor exceeds 256 characters, the leftmost 256 characters are provided. This field is right-padded with blanks as necessary.

The LICOPT specification, if of nonzero length, identifies an improperly specified Licensed Internal Code option intended for use in whole program optimization. This field is right-padded with blanks as necessary.

The reason code indicates the specific reason for the exception, and is one of the following:

Error

Code

Meaning

01

"ForbiddenClass": The class name was found to be used in the application, but is not permitted in a stand-alone Java program.

02

"ForbiddenMethod": The method name was found to be used in the application, but is not permitted in a stand-alone Java program.

03

"ClassNotFound": The class name was referenced in the classes to be analyzed in creating the stand-alone Java program, but could not be located.

04

"NoRootClasses": No classes were specified as containing external entry points into the stand-alone Java program.

05

"UserNativeMethod": The method name was found to be used in the application, but is a user native method, and the AllowUserNativeMethods Licensed Internal Code option was not specified.

06

"BadlyFormedLicopt": The LICOPT specification provided by the user contains a keyword recognized as being important to whole program optimization, but either the value is not in the correct form or the value refers to a class, method, or field that could not be located.

07

"SecurityCheck": An illegal manipulation of a Java program was detected and prevented.
