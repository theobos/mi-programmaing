<TR VALIGN="TOP"><TD><P><B>4AA1
</B></TD><TD><P><I>Java Class Format Error
</I>
<P>
An error was detected in the format of a Java class file.
<P>
<I>Information Passed:</I>

</TD><TD ALIGN="LEFT" VALIGN="TOP">

<P>The <B><U>class file pointer</U></B> addresses the vnode of
the Java class file or Java archive (JAR) file containing
the error being reported.
<P>If the <I>class file pointer</I> addresses the vnode
of a JAR file, the <B><U>class number</U></B> indicates, by index, which
class in the JAR file contains the error.  If the <I>class
file pointer</I> addresses the vnode of a Java class file, <I>class number</I> is zero.
<P>The <B><U>error offset</U></B> is the byte offset into the
class file or JAR file bytecode stream at which the error
occurs.
<P>The <B><U>reason code</U></B> indicates the specific reason
for the exception, and is one of the following&#58;
<TABLE CELLPADDING="3">
<TR VALIGN="TOP"><TD><P><B>Reason</B>
</TD><TD><P><B></B>
</TD></TR><TR VALIGN="TOP"><TD><P><B>Code</B>
</TD><TD><P><B>Meaning</B>
</TD></TR><TR VALIGN="TOP"><TD><P>00001001
</TD><TD><P>Truncated class file.
</TD></TR><TR VALIGN="TOP"><TD><P>00001002
</TD><TD><P>Extra data at end of class file.
</TD></TR><TR VALIGN="TOP"><TD><P>00001101
</TD><TD><P>The first 4 bytes of a class file must contain the value
hex CAFEBABE.
</TD></TR><TR VALIGN="TOP"><TD><P>00001201
</TD><TD><P><I>minor_version</I> is not valid.
</TD></TR><TR VALIGN="TOP"><TD><P>00001202
</TD><TD><P><I>major_version</I> is not valid.
</TD></TR><TR VALIGN="TOP"><TD><P>00001302
</TD><TD><P>Interface classes must not have the ACC_FINAL set in <I>access_flags</I>.
</TD></TR><TR VALIGN="TOP"><TD><P>00001304
</TD><TD><P>Classes cannot have both ACC_FINAL and ACC_ABSTRACT set in <I>access_flags</I>.
</TD></TR><TR VALIGN="TOP"><TD><P>00001411
</TD><TD><P><I>this_class</I> must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00001412
</TD><TD><P><I>this_class</I> must refer to a CONSTANT_Class constant pool entry
</TD></TR><TR VALIGN="TOP"><TD><P>00001413
</TD><TD><P>(Jar file members only) The package/class name of the Jar file
member must match the package/class name of the class within the file.
</TD></TR><TR VALIGN="TOP"><TD><P>00001421
</TD><TD><P>The value of <I>super_class</I> must be either 0 or a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00001422
</TD><TD><P><I>super_class</I> may only be 0 if the class file represents
a class (not an interface).
</TD></TR><TR VALIGN="TOP"><TD><P>00001423
</TD><TD><P>If <I>super_class</I> is non-zero then it must refer to a
CONSTANT_Class constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00001424
</TD><TD><P>If <I>super_class</I> is 0, then <I>this_class</I> must
refer to a CONSTANT_Class constant pool entry representing the class
'java.lang.Object'.
</TD></TR><TR VALIGN="TOP"><TD><P>00001426
</TD><TD><P><I>super_class</I> must not match <I>this_class</I>.
</TD></TR><TR VALIGN="TOP"><TD><P>00001501
</TD><TD><P>For class file attributes, the <I>attribute_name_index</I> must
be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00001502
</TD><TD><P>For class file attributes, <I>attribute_name_index</I>
must refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00001503
</TD><TD><P>There may be no more than one SourceFile attribute for the
class file.
</TD></TR><TR VALIGN="TOP"><TD><P>00002101
</TD><TD><P>The <I>constant_pool_count</I> must be greater than 0.
</TD></TR><TR VALIGN="TOP"><TD><P>00002102
</TD><TD><P>Constant pool tag values must be in the range from 1 to 12,
except for the value 2.
</TD></TR><TR VALIGN="TOP"><TD><P>00002201
</TD><TD><P>For a CONSTANT_Class constant pool entry, <I>name_index</I> must be a valid
constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002204
</TD><TD><P>For a CONSTANT_Class constant pool entry, the class name represented
by the CONSTANT_Utf8 entry (indicated by <I>name_index</I>)
cannot have more than 255 array dimensions.
</TD></TR><TR VALIGN="TOP"><TD><P>00002301
</TD><TD><P>For a CONSTANT_Fieldref constant pool entry, <I>class_index</I> must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002302
</TD><TD><P>For a CONSTANT_Fieldref constant pool entry, <I>class_index</I> must refer
to a CONSTANT_Class constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002304
</TD><TD><P>For a CONSTANT_Fieldref constant pool entry, <I>name_and_type_index</I>  must
be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002305
</TD><TD><P>For a CONSTANT_Fieldref constant pool entry, <I>name_and_type_index</I> must
refer to a CONSTANT_NameAndType constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002401
</TD><TD><P>For a CONSTANT_Methodref constant pool entry, <I>class_index</I> must be a
valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002402
</TD><TD><P>For a CONSTANT_Methodref constant pool entry, <I>class_index</I> must be
a CONSTANT_Class constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002404
</TD><TD><P>For a CONSTANT_Methodref constant pool entry, <I>name_and_type_index</I> must
be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002405
</TD><TD><P>For a CONSTANT_Methodref constant pool entry, <I>name_and_type_index</I> must
refer to a CONSTANT_NameAndType constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002406
</TD><TD><P>For a CONSTANT_Methodref constant pool entry, if the method
name identified by the CONSTANT_NameAndType constant pool entry
(indicated by <I>name_and_type_index</I>)
starts with '&lt;', then the full name must either be '&lt;init&gt;' or
'&lt;clinit&gt;'.
</TD></TR><TR VALIGN="TOP"><TD><P>00002407
</TD><TD><P>For a CONSTANT_Methodref constant pool entry, the special methods named '&lt;init&gt;' and
'&lt;clinit&gt;' may only return type 'void'.
</TD></TR><TR VALIGN="TOP"><TD><P>00002501
</TD><TD><P>For a CONSTANT_InterfaceMethodref constant pool entry, <I>class_index</I>
must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002502
</TD><TD><P>For a CONSTANT_InterfaceMethodref constant pool entry, <I>class_index</I>
must refer to a CONSTANT_Class constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002504
</TD><TD><P>For a CONSTANT_InterfaceMethodref constant pool entry, <I>name_and_type_index</I> must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002505
</TD><TD><P>For a CONSTANT_InterfaceMethodref constant pool entry, <I>name_and_type_index</I>
must refer to a CONSTANT_NameAndType constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002506
</TD><TD><P>For a CONSTANT_InterfaceMethodref constant pool entry, if the method
name identified by the CONSTANT_NameAndType constant pool entry
(indicated by <I>name_and_type_index</I>)
starts with '&lt;', then the full name must either be '&lt;init&gt;' or
'&lt;clinit&gt;'.
</TD></TR><TR VALIGN="TOP"><TD><P>00002507
</TD><TD><P>For a CONSTANT_InterfaceMethodref constant pool entry, the special methods named
'&lt;init&gt;' and '&lt;clinit&gt;' may only return type 'void'.
</TD></TR><TR VALIGN="TOP"><TD><P>00002601
</TD><TD><P>For a CONSTANT_String constant pool entry, <I>string_index</I> must be
a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002602
</TD><TD><P>For a CONSTANT_String constant pool entry, <I>string_index</I> must
refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002b01
</TD><TD><P>For a CONSTANT_NameAndType constant pool entry, <I>name_index</I>
must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002b02
</TD><TD><P>For a CONSTANT_NameAndType constant pool entry, <I>name_index</I> must
refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002b03
</TD><TD><P>For a CONSTANT_NameAndType constant pool entry, the name identified
by <I>name_index</I> must be a valid
Java field or method name, that is a simple (not qualified) name.
</TD></TR><TR VALIGN="TOP"><TD><P>00002b04
</TD><TD><P>For a CONSTANT_NameAndType constant pool entry, <I>descriptor_index</I>  must
be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00002b05
</TD><TD><P>For a CONSTANT_NameAndType constant pool entry, <I>descriptor_index</I> must
refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00002b06
</TD><TD><P>For a CONSTANT_NameAndType constant pool entry, the descriptor
identified by <I>descriptor_index</I> must be a
valid Java field or method descriptor.
</TD></TR><TR VALIGN="TOP"><TD><P>00002c01
</TD><TD><P>For a CONSTANT_Utf8 constant pool entry, no byte in the <I>bytes</I> array may
have the value hex 00 or a value in the range hex F0 to hex FF.
</TD></TR><TR VALIGN="TOP"><TD><P>00003001
</TD><TD><P>Each value in the <I>interfaces</I> array must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00003002
</TD><TD><P>Each constant pool index in the interfaces array, must refer to a CONSTANT_Class
constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00004001
</TD><TD><P>No two fields in a class file may have the same name and
descriptor.
</TD></TR><TR VALIGN="TOP"><TD><P>00004102
</TD><TD><P>Interface fields must have ACC_PUBLIC, ACC_STATIC, and ACC_FINAL
set in <I>access_flags</I>. No other flags are allowed for interface fields.
</TD></TR><TR VALIGN="TOP"><TD><P>00004103
</TD><TD><P>Class fields may have only one of flags ACC_PUBLIC,
ACC_PROTECTED, and ACC_PRIVATE set in <I>access_flags</I> (if any).
</TD></TR><TR VALIGN="TOP"><TD><P>00004104
</TD><TD><P>A field cannot have both ACC_FINAL and ACC_VOLATILE set in <I>access_flags</I>
</TD></TR><TR VALIGN="TOP"><TD><P>00004201
</TD><TD><P>For a field_info structure, <I>name_index</I>
must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00004202
</TD><TD><P>For a field_info structure, <I>name_index</I> must
refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00004203
</TD><TD><P>For a field_info structure, the name identified
by <I>name_index</I> must be a valid
Java field name, that is a simple (not qualified) name.
</TD></TR><TR VALIGN="TOP"><TD><P>00004301
</TD><TD><P>For a field_info structure, <I>descriptor_index</I>  must be a
valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00004302
</TD><TD><P>For a field_info structure, <I>descriptor_index</I> must
refer to CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00004303
</TD><TD><P>For a field_info structure, the descriptor identified
by <I>descriptor_index</I> must represent a
valid Java field descriptor.
</TD></TR><TR VALIGN="TOP"><TD><P>00004401
</TD><TD><P>For a field_info structure, <I>attribute_name_index</I>
must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00004402
</TD><TD><P>For a field_info structure, <I>attribute_name_index</I>
must refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00004404
</TD><TD><P>There can be no more than one ConstantValue attribute per
field_info structure.
</TD></TR><TR VALIGN="TOP"><TD><P>00005001
</TD><TD><P>No two methods in a class file may have the same name and
descriptor.
</TD></TR><TR VALIGN="TOP"><TD><P>00005002
</TD><TD><P>For an interface class, only the &lt;clinit&gt; method may be
implemented (have a Code attribute). All other methods must be
abstract methods.
</TD></TR><TR VALIGN="TOP"><TD><P>00005102
</TD><TD><P>Methods within an interface class must have ACC_PUBLIC and
ACC_ABSTRACT set in <I>access_flags</I>. Only these flags can be
set for interface methods.
</TD></TR><TR VALIGN="TOP"><TD><P>00005103
</TD><TD><P>Constructors (named '&lt;init&gt;') may only have ACC_PUBLIC,
ACC_PROTECTED, and ACC_PRIVATE set in <I>access_flags</I>.
</TD></TR><TR VALIGN="TOP"><TD><P>00005104
</TD><TD><P>For a method_info structure, at most, only one of ACC_PUBLIC,
ACC_PROTECTED and ACC_PRIVATE may be set in <I>access_flags</I>.
</TD></TR><TR VALIGN="TOP"><TD><P>00005105
</TD><TD><P>For a method_info structure, ACC_ABSTRACT may not be used together
with ACC_FINAL, ACC_NATIVE or ACC_SYNCHRONIZED in <I>access_flags</I>.
</TD></TR><TR VALIGN="TOP"><TD><P>00005201
</TD><TD><P>For a method_info structure, <I>name_index</I> must be a valid
constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00005202
</TD><TD><P>For a method_info structure, <I>name_index</I> must
refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00005203
</TD><TD><P>For a method_info structure, the name identified
by <I>name_index</I> must be one of the
special method names (either '&lt;init&gt;' or '&lt;clinit&gt;') or be valid
Java method name, that is a simple (not qualified) name.
</TD></TR><TR VALIGN="TOP"><TD><P>00005301
</TD><TD><P>For a method_info structure, <I>descriptor_index</I> must be a
valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00005302
</TD><TD><P>For a method_info structure, <I>descriptor_index</I> must
refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00005304
</TD><TD><P>For a method_info structure, the descriptor identified
by <I>descriptor_index</I> is limited to 255 words of parameters,
(including 1 word for 'this' in the case of instance methods).
Parameters of type 'long' and 'double' use 2 words each.
</TD></TR><TR VALIGN="TOP"><TD><P>00005401
</TD><TD><P>For a method_info structure, <I>attribute_name_index</I>
must be a valid constant pool index.
</TD></TR><TR VALIGN="TOP"><TD><P>00005402
</TD><TD><P>For a method_info structure, <I>attribute_name_index</I> must
refer to a CONSTANT_Utf8 constant pool entry.
</TD></TR><TR VALIGN="TOP"><TD><P>00005403
</TD><TD><P>For a method_info structure, if the <I>access_flags</I> do not
indicate that the method is native or abstract, then there must be
exactly one Code attribute.
</TD></TR><TR VALIGN="TOP"><TD><P>00005404
</TD><TD><P>For a method_info structure, there may be at most one Exceptions
attribute.
</TD></TR><TR VALIGN="TOP"><TD><P>00005406
</TD><TD><P>For a method_info structure, if the <I>access_flags</I> indicate
that the method is native or abstract, there must be no Code
attribute.
</TD></TR><TR VALIGN="TOP"><TD><P>00006101
</TD><TD><P>For a SourceFile attribute, <I>attribute_length</I> must be 2.
</TD></TR><TR VALIGN="TOP"><TD><P>00006102
